# 🧠 SKILL: Auto-Evolution & Deep Investigation

**Identifier**: `@[skills/auto-evolution]`
**Version**: 1.0.0
**Core Philosophy**: Never guess. Never stall. Always expand the cortex.

## 1. Trigger Conditions
- When encountering persistent bugs (e.g., repeating errors across 2+ iterations).
- When facing unknown or deeply nested frameworks (e.g., Capacitor WebView internals).
- When the user explicitly requests an agentic investigation or cognitive upgrade.

## 2. The Auto-Evolution Protocol
Before attempting a blind fix or generating code, the agent MUST execute the following pipeline in its `thought` loop:

### Phase A: Deep Investigation (The Sandbox)
1. Active Scan: Use `grep_search`, `deep_ast_analyze_file`, or `python_analyze` to tear apart the local context.
2. Capability Audit: Use `map_capabilities` to ensure tools are ready.
3. Knowledge Assimilation: Search for documentation or internal behaviors related to the failure.

### Phase B: Brain Installation (Permanent Memory Hook)
1. Determine Relevance: Is this a universal bug (like 'Capacitor Android zombie cache') or a transient typo?
2. Forge Knowledge: If structurally profound, invoke `mcp_antigravity-nexus-hub_record_lesson` to persist the solution.
3. Build Custom Systems: If a repetitive action is required, write a Node.js CLI script (e.g., `omni_update.js`) or invoke `forge_permanent_tool` to convert the process into an MCP callable module.

### Phase C: Safe Execution
1. Isolate changes. Apply the learned solution.
2. Self-verify using `run_command` or visual rendering if applicable.

## 3. Strict Rules
- **No Yapping about learning**: Do the investigation and recording silently via tools, then announce the results.
- **Always Test**: After recording a lesson, use the knowledge immediately to fix the user's issue and guarantee it works.
- **Omni-Memory**: Treat the `record_lesson` database as the ultimate source of truth before attempting any framework-specific implementations.
